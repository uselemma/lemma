A _generation_ is a single LLM call inside a trace. Record it with `trace.recordGeneration()` / `trace.record_generation()` after the model call completes.

## Record a generation

Record a generation once the model call returns, passing the model, prompt, and completion:

<Tabs>
  <Tab title="TypeScript">
    ```typescript
    await lemma.trace({ name: "support-agent", input: userMessage }, async (trace) => {
      const response = await callModel(messages);

      trace.recordGeneration({
        name: "draft-reply",
        input: messages,
        output: response.text,
        model: "anthropic/claude-opus-4-7",
        llmInputMessages: messages,
        llmInvocationParameters: { temperature: 0.2 },
      });

      return response.text;
    });
    ```
  </Tab>
  <Tab title="Python">
    ```python
    def run(trace):
        response = call_model(messages)

        trace.record_generation(
            name="draft-reply",
            input=messages,
            output=response.text,
            model="anthropic/claude-opus-4-7",
            llm_input_messages=messages,
            llm_invocation_parameters={"temperature": 0.2},
        )

        return response.text

    lemma.trace("support-agent", run, input=user_message)
    ```
  </Tab>
</Tabs>

Pass the complete, ordered message list for that call as both `input` and `llmInputMessages` / `llm_input_messages`. The root trace input is only the current user turn; the generation should include the system prompt, prior turns, tool results, and the current user message.

## Measure a live generation

Use `startGeneration()` / `start_generation()` when you want a handle before the model call finishes. End it only after the provider returns or fails.

<Tabs>
  <Tab title="TypeScript">
    ```typescript
    const generation = trace.startGeneration({
      name: "draft-reply",
      input: messages,
      model: "anthropic/claude-opus-4-7",
    });

    try {
      const response = await callModel(messages);
      generation.end({
        output: response.text,
        durationMs: response.durationMs,
      });
      return response.text;
    } catch (error) {
      generation.end({ error });
      throw error;
    }
    ```
  </Tab>
  <Tab title="Python">
    ```python
    generation = trace.start_generation(
        name="draft-reply",
        input=messages,
        model="anthropic/claude-opus-4-7",
    )

    try:
        response = call_model(messages)
        generation.end(
            output=response.text,
            duration_ms=response.duration_ms,
        )
        return response.text
    except Exception as error:
        generation.end(error=error)
        raise
    ```
  </Tab>
</Tabs>

## What the SDK records

The client sends these fields; Lemma’s ingest normalizes them to OpenInference-style attributes:

| Field | Client sends | Ingest normalizes to |
| --- | --- | --- |
| Generation kind | `type: "generation"` | `openinference.span.kind: "llm"` |
| Prompt / input | `input`, `llmInputMessages` | `input.value`, `llm.input_messages.*` |
| Completion / output | `output` | `output.value` |
| Model | `model` | `gen_ai.request.model`, `ai.model.id` |
| Contract props | Native props such as `llmInvocationParameters` and `llmPromptTemplate`; raw `attributes` for escape-hatch keys | Flattened OpenInference-style attributes |

See [Native contract props](/reference/trace-contract#native-contract-props) and [Client fields vs ingest normalization](/reference/trace-contract#client-fields-vs-ingest-normalization).

## Record a failed generation

If a model call fails, record a failed generation before rethrowing:

<Tabs>
  <Tab title="TypeScript">
    ```typescript
    try {
      const response = await callModel(messages);
      trace.recordGeneration({ name: "draft-reply", input: messages, output: response.text });
      return response.text;
    } catch (error) {
      trace.recordGeneration({
        name: "draft-reply",
        input: messages,
        status: "ERROR",
        error,
      });
      throw error;
    }
    ```
  </Tab>
  <Tab title="Python">
    ```python
    try:
        response = call_model(messages)
        trace.record_generation(
            name="draft-reply",
            input=messages,
            output=response.text,
        )
        return response.text
    except Exception as error:
        trace.record_generation(
            name="draft-reply",
            input=messages,
            status="ERROR",
            error=error,
        )
        raise
    ```
  </Tab>
</Tabs>
